# LINE × AI 自動応答ボット（GAS版）構築手順

AIプログラミングセミナー受講生の自習フォロー用LINEボットの最小構成（MVP）です。
LINEで送ったテキストをGemini APIに投げて、優しく返してくれます。

---

## 全体像

```
ユーザー ──LINE送信──▶ LINE Platform ──Webhook──▶ GAS(doPost)
                                                   │
                                                   └─Gemini API──▶ 回答生成
                                                   │
ユーザー ◀──LINEで表示── LINE Platform ◀──返信──── GAS
```

所要時間：だいたい20〜30分。

---

## 事前に用意するもの

- LINEアカウント（個人のものでOK）
- Googleアカウント（GAS用）
- そのへんで作業するブラウザ

---

## ステップ1：LINEの「公式アカウント＋チャネル」を作る

### 1-1. LINE Developersにログイン

1. https://developers.line.biz/ja/ にアクセス
2. 右上「コンソール」→ LINEアカウントでログイン
3. 初回のみ「開発者名」「メール」を登録

### 1-2. プロバイダーを作る

1. 「プロバイダー」欄の【作成】をクリック
2. 名前は自由（例：`セミナーボット`）

### 1-3. Messaging APIチャネルを作る

1. 作ったプロバイダーを開いて【新規チャネル作成】
2. 種類で **「Messaging API」** を選択
3. 以下を入力：
   - チャネル名：例 `セミナーAIアシスタント`
   - チャネル説明：自由
   - 大業種・小業種：自由
   - 利用規約に同意して【作成】

### 1-4. チャネルアクセストークンを発行

1. 作成したチャネルを開く
2. 上部タブ **「Messaging API設定」** を開く
3. ページ一番下「**チャネルアクセストークン（長期）**」の【発行】をクリック
4. 出てきた長い文字列を **メモ帳などにコピー** しておく
   → あとで `Code.gs` の `LINE_ACCESS_TOKEN` に貼ります

### 1-5. 余計な自動応答をOFFにする

同じ「Messaging API設定」ページの下のほうに「LINE公式アカウント機能」の項目があるので、
リンクから公式アカウントマネージャーに飛んで以下を変更：

- **応答メッセージ**：オフ
- **あいさつメッセージ**：オフ（任意）
- **Webhook**：オン

> これをやらないと、デフォルトの「ご返信ありがとうございます…」が一緒に届いてしまいます。

## ステップ2：Gemini APIキーを取る（無料）

1. https://aistudio.google.com/app/apikey にアクセス
2. Googleアカウントでログイン
3. 【Create API key】→ 適当なプロジェクト名（または既存）を選んで作成
4. 表示されたキー（`AIza...`で始まる文字列）を **コピー** してメモしておく
   → あとで `Code.gs` の `GEMINI_API_KEY` に貼ります

> Gemini APIは無料枠だけでも十分試せます。

---

## ステップ3：Google Apps Script（GAS）を作る

### 3-1. GASプロジェクトを新規作成

1. https://script.google.com/ にアクセス
2. 左上【新しいプロジェクト】
3. プロジェクト名（左上の「無題のプロジェクト」をクリック）を `LINE-AI-Bot` などに

### 3-2. コードを貼り付ける

1. 左ペインで `コード.gs`（または `Code.gs`）を開く
2. 中身をぜんぶ消して、本リポジトリの **[`Code.gs`](./Code.gs) の中身をまるごとコピペ**

### 3-3. 設定エリアを書き換える

ファイル冒頭の2行を、自分のキーに置き換えます：

```javascript
const LINE_ACCESS_TOKEN = 'ここにLINEのチャネルアクセストークンを貼り付け';
const GEMINI_API_KEY    = 'ここにGeminiのAPIキーを貼り付け';
```

シングルクォート `'` の **中だけ** 書き換えるのがポイント。

### 3-4. 保存

`Ctrl + S`（Macは `Cmd + S`）またはフロッピーアイコンで保存。

### 3-5. （任意）Gemini単体で動作確認

1. 関数選択プルダウンで `testGemini` を選び【実行】
2. 初回は「権限を確認」→ 自分のアカウントを選択 → 「詳細」→「(プロジェクト名)に移動」→「許可」
3. 実行ログにAIの返事が出ればGemini側はOK

---

## ステップ4：GASを「ウェブアプリ」としてデプロイ（公開）

1. 右上【デプロイ】→【新しいデプロイ】
2. 左上の歯車アイコン → **「ウェブアプリ」** を選択
3. 設定を以下にして【デプロイ】：

   | 項目 | 値 |
   |---|---|
   | 説明 | （任意）`v1` など |
   | 次のユーザーとして実行 | **自分** |
   | アクセスできるユーザー | **全員** |

4. 表示された **「ウェブアプリのURL」** をコピー
   - `https://script.google.com/macros/s/xxxxxxx/exec` のような形式

> ⚠ 「アクセスできるユーザー」を **全員** にしないと、LINEからのリクエストが届きません。

---

## ステップ5：LINEのWebhookにGAS URLを設定

1. LINE Developersのチャネル→「Messaging API設定」タブに戻る
2. **「Webhook URL」** の【編集】に、さっきコピーしたGAS URLを貼り付け→【更新】
3. その下の **「Webhookの利用」** を **ON** に
4. 【検証】ボタンを押して **「成功」** と出ればOK

> 「エラー」が出る場合：GASのデプロイ設定で「全員」になっていないことが多いです。

---

## ステップ6：友だち追加して話しかける

1. 「Messaging API設定」タブの上部にあるQRコードをLINEで読み取って **友だち追加**
2. 何かメッセージを送ってみる
3. AIから返事が来たら完成！🎉

### おすすめのお試しメッセージ

- `Pythonでファイル読み込みでエラーが出ました`
- `TypeError: Cannot read properties of undefined`
- `for文の書き方を教えて`

---

## うまく動かないときのチェックリスト

| 症状 | 確認ポイント |
|---|---|
| 返事が来ない | LINE公式アカウントマネージャーで「応答モード = Bot」「Webhook = ON」になっているか |
| 「ご返信ありがとうございます」が混ざる | 「応答メッセージ」をオフに |
| GAS実行ログにエラー | `LINE_ACCESS_TOKEN` と `GEMINI_API_KEY` のコピペミス（前後の空白に注意） |
| Webhook検証で失敗 | GASを「全員」公開で再デプロイしたか。コードを直したら **【デプロイの管理】→ 編集（鉛筆）→ バージョン「新バージョン」で更新** が必要 |
| 急に動かなくなった | コードを直したあと **再デプロイしないと反映されません** |

## 次の一歩（スケボーから自転車へ）

MVPが動いたら、こんな改善が考えられます：

- 会話履歴を `PropertiesService` に保存して、文脈をつなぐ
- 画像メッセージ対応（エラースクショをそのまま読ませる → Geminiの画像入力）
- ユーザーごとの利用回数制限・スプレッドシートへのログ保存
- 「先生に通知」コマンド（特定キーワードでメール送信）
